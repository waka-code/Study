# Single-AZ Deployment

## Descripción
- La base de datos se implementa en una sola Zona de Disponibilidad (AZ).
- Es la opción más económica.

## Casos de uso
- Entornos de desarrollo y pruebas.
- Aplicaciones no críticas.

## Limitaciones
- No ofrece alta disponibilidad.
- En caso de fallo de la AZ, la base de datos no estará disponible hasta que se restaure manualmente.
