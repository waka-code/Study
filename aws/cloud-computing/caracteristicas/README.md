# Características del Cloud Computing

## 1. Autoservicio bajo demanda (On-demand Self-Service)
El usuario puede provisionar recursos por sí mismo (como servidores, almacenamiento o bases de datos) sin necesidad de pedirlos a un técnico.

**Ejemplo:** Crear una máquina virtual en AWS con un par de clics.

---

## 2. Amplio acceso a la red (Broad Network Access)
Los servicios de la nube se pueden acceder desde cualquier lugar con conexión a internet y desde múltiples dispositivos: laptops, móviles, tablets, etc.

**Ejemplo:** Usar un dashboard de Azure desde tu laptop o desde tu celular.

---

## 3. Alquiler múltiple y agrupación de recursos (Resource Pooling / Multitenancy)
Los recursos físicos (CPU, RAM, almacenamiento, red) se comparten entre múltiples clientes, pero cada uno aislado y seguro. El proveedor de la nube agrupa recursos y los asigna dinámicamente según demanda.

**Ejemplo:** AWS usa los mismos servidores físicos para varios clientes, pero cada uno está virtualmente separado.

---

## 4. Rápida elasticidad y escalabilidad (Rapid Elasticity)
La nube permite aumentar o disminuir recursos de forma rápida y automática según la demanda.

**Ejemplo:** Una aplicación web puede duplicar la cantidad de servidores durante un pico de tráfico y reducirlos cuando baja.

---

## 5. Servicio medido (Measured Service)
El uso de los recursos se mide constantemente (CPU, ancho de banda, almacenamiento, instancias activas) y el cliente paga solo por lo que consume.

**Ejemplo:** Si una base de datos usó 20 GB este mes, solo se cobra esos 20 GB.
