# Modelos de Cloud Computing

## ☁️ Cloud privado
Un **cloud privado** es una infraestructura en la nube que pertenece y es utilizada exclusivamente por una sola organización. Puede estar alojada en las instalaciones de la empresa (on-premise) o en un centro de datos externo, pero solo esa empresa tiene acceso.

**Características clave:**
- Máximo control sobre seguridad, configuración y datos.
- Ideal para organizaciones con requisitos estrictos de cumplimiento (bancos, gobiernos, hospitales).
- Costos más altos porque la empresa mantiene o paga toda la infraestructura.

---

## ☁️ Cloud público
El **cloud público** es un modelo en el que los recursos de computación (servidores, almacenamiento, redes) se ofrecen al público general a través de un proveedor externo, como AWS, Azure o Google Cloud.

**Características clave:**
- La infraestructura es compartida entre múltiples clientes.
- Escalable y accesible bajo demanda.
- Pago por uso.
- Menor costo y cero mantenimiento físico, ya que el proveedor lo administra.

---

## ☁️ Cloud híbrido
El **cloud híbrido** combina un cloud privado con un cloud público, permitiendo mover datos y aplicaciones entre ambos según convenga.

**Características clave:**
- Flexibilidad para usar lo mejor de ambos mundos.
- Se mantiene información sensible en el cloud privado y se aprovecha el cloud público para carga variable o picos de demanda.
- Ideal cuando necesitas control y seguridad, pero sin renunciar a escalabilidad.
