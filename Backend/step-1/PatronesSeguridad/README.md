# Patrones de Seguridad en Backend

Esta carpeta contiene los principales patrones y prácticas de seguridad para backend profesional. Cada punto tiene su propio README y ejemplo.

## Núcleo obligatorio
- Authentication vs Authorization
- Principle of Least Privilege
- Input Validation & Sanitization
- Password Hashing
- JWT / Sessions / Tokens
- SQL Injection & XSS (cómo prevenirlos)
- Error handling seguro (Fail Securely)

## Nivel Senior fuerte
- Defense in Depth
- Rate Limiting
- CSRF Protection
- Secure Secrets Management
- Audit Logging
- CORS vs CSP
- HTTPS / TLS

## Nivel Senior Arquitecto (bonus)
- Zero Trust
- Circuit Breaker
- WAF / Cloudflare
- Security Headers
- DevSecOps (scanning, CI/CD)

> Si fallas en el núcleo, no hay senior, aunque sepas frameworks.
