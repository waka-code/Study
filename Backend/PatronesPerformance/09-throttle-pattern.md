# Throttle Pattern

Permite ejecutar una acción a intervalos definidos, ignorando la frecuencia de eventos. Útil para limitar la frecuencia de actualizaciones o llamadas.

**Ventajas:**
- Controla el ritmo de ejecución.
- Evita saturación por eventos frecuentes.

**Trade-off:**
- Puede perder eventos importantes.
- Requiere balance entre frecuencia y necesidad.
