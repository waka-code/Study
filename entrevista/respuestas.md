# Preguntas de Entrevista con Respuestas

> Aquí puedes ir completando las respuestas a cada pregunta según tu experiencia y el nivel requerido.

## Ejemplo:

### ¿Qué es una variable?
Una variable es un espacio en memoria donde se almacena un valor que puede cambiar durante la ejecución del programa.

### ¿Qué es una función?
Una función es un bloque de código reutilizable que realiza una tarea específica y puede recibir parámetros y devolver un resultado.

### ¿Qué es una clase?
Una clase es un molde o plantilla para crear objetos que comparten atributos y métodos.

### ¿Qué es un array?
Una estructura de datos que almacena una colección ordenada de elementos, accesibles por índice.

### ¿Qué es un objeto?
Una estructura que agrupa datos y funciones relacionadas bajo una misma entidad, usando pares clave-valor.

### ¿Qué es herencia?
Un mecanismo de la programación orientada a objetos donde una clase puede heredar atributos y métodos de otra.

### ¿Qué es polimorfismo?
La capacidad de un objeto de tomar muchas formas, permitiendo que una misma interfaz se use para diferentes tipos de datos.

### ¿Qué es encapsulamiento?
El principio de ocultar los detalles internos de un objeto y exponer solo lo necesario a través de métodos públicos.

### ¿Qué es una promesa?
Un objeto en JavaScript que representa la eventual finalización (o falla) de una operación asíncrona.

### ¿Qué es async/await?
Sintaxis de JavaScript para trabajar con promesas de forma más legible y estructurada, parecida a código síncrono.

### ¿Qué es una API?
Una interfaz que permite la comunicación entre diferentes sistemas o componentes de software.

### ¿Qué es HTTP?
Protocolo de comunicación usado para transferir datos en la web entre clientes y servidores.

### ¿Qué es REST?
Un estilo de arquitectura para diseñar servicios web que usan HTTP y operaciones estándar como GET, POST, PUT y DELETE.

### ¿Qué es JSON?
Un formato ligero de intercambio de datos, fácil de leer y escribir para humanos y máquinas.

### ¿Qué es una base de datos?
Un sistema organizado para almacenar, gestionar y recuperar información de forma eficiente.

### ¿Qué es SQL?
Lenguaje de consulta estructurado usado para interactuar con bases de datos relacionales.

### ¿Qué es NoSQL?
Tipo de base de datos no relacional, diseñada para manejar grandes volúmenes de datos no estructurados o semiestructurados.

### ¿Qué es un commit?
Una instantánea de los cambios realizados en el código, registrada en el historial de un sistema de control de versiones como Git.

### ¿Qué es Git?
Un sistema de control de versiones distribuido que permite gestionar el historial de cambios en el código fuente.

### ¿Qué es un branch?
Una rama en el historial de Git que permite desarrollar funcionalidades de forma aislada antes de fusionarlas al proyecto principal.

### ¿Qué es merge?
La acción de combinar los cambios de dos ramas diferentes en una sola.

### ¿Qué es un conflicto de merge?
Ocurre cuando Git no puede combinar automáticamente los cambios de dos ramas y requiere intervención manual.

### ¿Qué es un bug?
Un error o defecto en el software que causa un comportamiento inesperado o incorrecto.

### ¿Qué es un test unitario?
Una prueba automatizada que verifica el funcionamiento correcto de una unidad mínima de código, como una función o método.

### ¿Qué es un endpoint?
Una URL específica de una API donde se puede acceder a un recurso o funcionalidad.

### ¿Qué es una petición GET/POST/PUT/DELETE?
Operaciones estándar de HTTP para leer, crear, actualizar o eliminar recursos en una API.

### ¿Qué es un error 404/500?
Códigos de estado HTTP: 404 indica recurso no encontrado, 500 indica error interno del servidor.

### ¿Qué es una cookie?
Pequeño archivo almacenado en el navegador para guardar información de sesión o preferencias del usuario.

### ¿Qué es una sesión?
Un mecanismo para mantener el estado de un usuario a lo largo de múltiples peticiones en una aplicación web.

### ¿Qué es CORS?
Política de seguridad que controla qué recursos pueden ser solicitados desde otros dominios en una aplicación web.

### ¿Qué es un token?
Un valor único usado para autenticar o autorizar a un usuario en una aplicación.

### ¿Qué es JWT?
(JSON Web Token) Un estándar para transmitir información de forma segura entre partes como un objeto JSON firmado.

### ¿Qué es HTTPS?
Versión segura de HTTP que cifra la comunicación entre cliente y servidor usando TLS/SSL.

### ¿Qué es un framework?
Conjunto de herramientas y librerías que facilitan el desarrollo de aplicaciones siguiendo ciertas reglas y estructuras.

### ¿Qué es una librería?
Colección de funciones o clases reutilizables que se pueden incorporar en un proyecto para añadir funcionalidad.

### ¿Qué es un IDE?
(Entorno de Desarrollo Integrado) Aplicación que reúne herramientas para programar, como editor, depurador y gestor de proyectos.

### ¿Qué es una variable de entorno?
Valor externo al código fuente que configura el comportamiento de una aplicación, como claves o rutas.

### ¿Qué es un log?
Registro de eventos, errores o información relevante generada por una aplicación durante su ejecución.

### ¿Qué es una excepción?
Un evento que interrumpe el flujo normal de un programa debido a un error o condición inesperada.

### ¿Qué es una interfaz?
Una definición de métodos y propiedades que una clase debe implementar, sin especificar su comportamiento.

### ¿Qué es un hook?
En React, una función especial que permite usar estado y otras características en componentes funcionales.

### ¿Qué es un componente?
Unidad reutilizable de interfaz de usuario en frameworks como React, Angular o Vue.

### ¿Qué es el DOM?
(Document Object Model) Representación estructurada de los elementos de una página web que puede ser manipulada con JavaScript.

### ¿Qué es un SPA?
(Single Page Application) Aplicación web que carga una sola página HTML y actualiza dinámicamente el contenido sin recargar la página completa.

### ¿Qué es SSR?
(Server-Side Rendering) Técnica donde el HTML se genera en el servidor y se envía listo al navegador.

### ¿Qué es CSR?
(Client-Side Rendering) Técnica donde el HTML se genera en el navegador usando JavaScript.

### ¿Qué es lazy loading?
Cargar recursos (imágenes, componentes, módulos) solo cuando se necesitan para mejorar el rendimiento.

### ¿Qué es memoization?
Técnica para guardar el resultado de funciones costosas y evitar recalcular si los datos no cambian.

### ¿Qué es una función pura?
Función que, dado un mismo input, siempre devuelve el mismo output y no tiene efectos secundarios.

### ¿Qué es inmutabilidad?
Propiedad de los datos que no pueden ser modificados después de ser creados.

### ¿Qué es una closure?
Función que recuerda el entorno en el que fue creada, incluso después de que ese entorno haya terminado de ejecutarse.

### ¿Qué es recursividad?
Técnica donde una función se llama a sí misma para resolver un problema dividiéndolo en subproblemas más pequeños.

### ¿Qué es un callback?
Función que se pasa como argumento a otra función y se ejecuta después de que ocurre un evento o se completa una tarea.

### ¿Qué es debounce/throttle?
Técnicas para limitar la frecuencia con la que se ejecuta una función en respuesta a eventos repetidos.

### ¿Qué es una expresión regular?
Patrón que describe un conjunto de cadenas y se usa para buscar o manipular texto.

### ¿Qué es un hash?
Función que transforma datos de tamaño variable en un valor de tamaño fijo, usado para búsquedas rápidas o seguridad.

### ¿Qué es una clave primaria/foránea?
Clave primaria identifica de forma única un registro en una tabla; clave foránea referencia a la clave primaria de otra tabla para establecer relaciones.

### ¿Qué es normalización?
Proceso de organizar los datos en una base de datos para reducir redundancia y mejorar la integridad.

### ¿Qué es un índice?
Estructura que acelera las búsquedas y consultas en una base de datos.

### ¿Qué es una transacción?
Conjunto de operaciones que se ejecutan como una sola unidad lógica, garantizando atomicidad, consistencia, aislamiento y durabilidad (ACID).

### ¿Qué es ACID?
Conjunto de propiedades (Atomicidad, Consistencia, Aislamiento, Durabilidad) que garantizan la fiabilidad de las transacciones en bases de datos.

### ¿Qué es CAP?
Teorema que establece que una base de datos distribuida solo puede garantizar dos de estas tres propiedades: Consistencia, Disponibilidad y Tolerancia a particiones.

### ¿Qué es sharding?
Dividir los datos en fragmentos (shards) distribuidos en varios servidores para escalar horizontalmente.

### ¿Qué es una réplica?
Copia de una base de datos que se usa para balancear lecturas y mejorar disponibilidad.

### ¿Qué es escalado vertical/horizontal?
Vertical: aumentar recursos de un solo servidor. Horizontal: agregar más servidores para repartir la carga.

### ¿Qué es caching?
Almacenar datos en memoria para acelerar el acceso y reducir la carga en sistemas de origen.

### ¿Qué es CDN?
(Red de entrega de contenido) Sistema distribuido de servidores que entrega contenido web de forma rápida y eficiente según la ubicación del usuario.

### ¿Qué es un load balancer?
Dispositivo o software que distribuye el tráfico entre varios servidores para optimizar recursos y disponibilidad.

### ¿Qué es Docker?
Plataforma para crear, desplegar y ejecutar aplicaciones en contenedores portables y aislados.

### ¿Qué es un contenedor?
Unidad ligera y portable que empaqueta una aplicación y sus dependencias para ejecutarse de forma consistente en cualquier entorno.

### ¿Qué es Kubernetes?
Sistema de orquestación de contenedores que automatiza el despliegue, escalado y gestión de aplicaciones en contenedores.

### ¿Qué es CI/CD?
Práctica de integración y entrega/despliegue continuo de código, automatizando pruebas y despliegues.

### ¿Qué es un pipeline?
Secuencia automatizada de pasos para construir, probar y desplegar código.

### ¿Qué es un test de integración?
Prueba que verifica la interacción entre varios módulos o servicios de una aplicación.

### ¿Qué es un test E2E?
Prueba el sistema completo como lo haría un usuario real, de extremo a extremo.

### ¿Qué es contract testing?
Verifica que la comunicación entre servicios (APIs) cumple el contrato esperado.

### ¿Qué es observabilidad?
Capacidad de medir el estado interno de un sistema a partir de sus salidas externas, usando logs, métricas y trazas.

### ¿Qué es logging/metrics/tracing?
Logging: registro de eventos. Metrics: medición de valores numéricos. Tracing: seguimiento de una petición a través de servicios.

### ¿Qué es una alerta?
Notificación automática ante errores o métricas anómalas en un sistema.

### ¿Qué es un microservicio?
Estilo de arquitectura donde una aplicación se divide en pequeños servicios independientes que se comunican entre sí.

### ¿Qué es monolito?
Aplicación donde toda la funcionalidad está en un solo bloque de código y despliegue.

### ¿Qué es DDD?
(Domain-Driven Design) Enfoque de diseño de software centrado en el dominio del negocio y su lógica.

### ¿Qué es un bounded context?
Límite explícito dentro del cual un modelo de dominio es definido y aplicable.

### ¿Qué es un value object?
Objeto que representa un valor y no tiene identidad propia, definido solo por sus atributos.

### ¿Qué es un aggregate?
Conjunto de entidades y objetos de valor tratados como una unidad de consistencia en DDD.

### ¿Qué es un repository?
Patrón que encapsula la lógica de acceso a datos para los aggregates.

### ¿Qué es un domain event?
Evento que representa algo que ocurrió en el dominio y puede ser relevante para otros componentes.

### ¿Qué es Clean Architecture?
Arquitectura que separa el dominio de la aplicación de los detalles externos, facilitando el mantenimiento y la escalabilidad.

### ¿Qué es Hexagonal Architecture?
Arquitectura que aísla el core de la aplicación de detalles externos mediante puertos y adaptadores.

### ¿Qué es una capa de infraestructura?
Capa encargada de la interacción con sistemas externos como bases de datos, servicios o frameworks.

### ¿Qué es una capa de dominio?
Capa que contiene la lógica de negocio pura y las reglas del dominio.

### ¿Qué es una capa de aplicación?
Capa que orquesta los casos de uso y coordina el flujo entre UI y dominio.

### ¿Qué es una interfaz de puerto/adaptador?
Contrato que define cómo interactúan los detalles externos con el core de la aplicación en arquitecturas como Hexagonal.

### ¿Qué es una API RESTful?
API que sigue los principios de REST, usando HTTP y recursos bien definidos.

### ¿Qué es GraphQL?
Lenguaje de consulta para APIs que permite a los clientes especificar exactamente qué datos necesitan.

### ¿Qué es WebSocket?
Protocolo que permite comunicación bidireccional y en tiempo real entre cliente y servidor.

### ¿Qué es OAuth2?
Protocolo de autorización que permite a aplicaciones acceder a recursos de usuario sin compartir credenciales.

### ¿Qué es rate limiting?
Técnica para limitar la cantidad de peticiones que un usuario puede hacer en un periodo de tiempo.

### ¿Qué es XSS?
(Cross-Site Scripting) Vulnerabilidad que permite inyectar scripts maliciosos en páginas web vistas por otros usuarios.

### ¿Qué es CSRF?
(Cross-Site Request Forgery) Ataque donde un usuario autenticado es engañado para ejecutar acciones no deseadas.

### ¿Qué es SQL Injection?
Ataque donde se insertan comandos SQL maliciosos en una consulta para manipular la base de datos.

### ¿Qué es OWASP Top 10?
Lista de los 10 riesgos de seguridad más críticos para aplicaciones web.

### ¿Qué es gestión de secretos?
Práctica de almacenar y manejar claves, contraseñas y tokens de forma segura, fuera del código fuente.

### ¿Qué es un despliegue blue/green?
Estrategia de despliegue donde se mantienen dos entornos (blue y green) y el tráfico se cambia de uno a otro para minimizar downtime.

### ¿Qué es rollback?
Revertir una aplicación o base de datos a un estado anterior tras un fallo en producción.

### ¿Qué es feature toggle?
Técnica para activar o desactivar funcionalidades en producción sin desplegar nuevo código.

### ¿Qué es un circuito breaker?
Patrón que previene llamadas repetidas a un sistema que está fallando, permitiendo su recuperación.

### ¿Qué es idempotencia?
Propiedad de una operación que puede ejecutarse varias veces sin cambiar el resultado más allá de la primera vez.

### ¿Qué es eventual consistency?
Modelo donde los datos pueden no estar sincronizados inmediatamente, pero eventualmente todos los nodos tendrán el mismo estado.

### ¿Qué es una saga?
Patrón para gestionar transacciones distribuidas dividiéndolas en pasos locales con compensaciones en caso de fallo.

### ¿Qué es resiliencia?
Capacidad de un sistema para recuperarse de fallos y seguir funcionando correctamente.

### ¿Qué es tolerancia a fallos?
Habilidad de un sistema para seguir operando correctamente incluso si una parte falla.

### ¿Qué es un SLA?
(Service Level Agreement) Acuerdo formal sobre el nivel de servicio esperado entre proveedor y cliente.

### ¿Qué es un SLO?
(Service Level Objective) Objetivo específico y medible dentro de un SLA.

### ¿Qué es un SLI?
(Service Level Indicator) Métrica usada para medir el cumplimiento de un SLO.

### ¿Qué es un test de carga?
Prueba que evalúa el rendimiento de un sistema bajo una cantidad creciente de usuarios o peticiones.

### ¿Qué es un test de estrés?
Prueba que evalúa el comportamiento de un sistema bajo condiciones extremas o fuera de lo esperado.

### ¿Qué es un mock/stub/spy?
Mock: objeto simulado para pruebas. Stub: objeto con respuestas predefinidas. Spy: objeto que registra cómo fue usado.

### ¿Qué es inyección de dependencias?
Patrón que consiste en proporcionar las dependencias de un objeto desde fuera, en vez de crearlas internamente.

### ¿Qué es un patrón de diseño?
Solución reutilizable a un problema común de diseño de software.

### ¿Qué es Singleton/Factory/Adapter/Strategy/Observer/Command/State/Builder/Abstract Factory/Decorator/Facade?
Patrones de diseño que resuelven problemas comunes de creación, estructura y comportamiento de objetos en software.

### ¿Qué es SOLID?
Conjunto de cinco principios para escribir código orientado a objetos mantenible y escalable.

### ¿Qué es el principio de responsabilidad única?
Cada clase debe tener una sola razón para cambiar.

### ¿Qué es el principio abierto/cerrado?
Las entidades de software deben estar abiertas para extensión, pero cerradas para modificación.

### ¿Qué es el principio de sustitución de Liskov?
Las clases derivadas deben poder sustituir a sus clases base sin alterar el funcionamiento del programa.

### ¿Qué es el principio de segregación de interfaces?
Los clientes no deben depender de interfaces que no usan.

### ¿Qué es el principio de inversión de dependencias?
Los módulos de alto nivel no deben depender de módulos de bajo nivel, ambos deben depender de abstracciones.

### ¿Qué es un anti-patrón?
Solución comúnmente usada que en realidad es ineficaz o contraproducente.

### ¿Qué es un test flaky?
Prueba automatizada que falla o pasa de forma intermitente sin cambios en el código.

### ¿Qué es Playwright?
Herramienta moderna para pruebas E2E que permite automatizar navegadores y validar aplicaciones web.

### ¿Qué es un entorno productivo/staging/dev?
Productivo: entorno real de usuarios. Staging: entorno de pruebas similar a producción. Dev: entorno de desarrollo.

### ¿Qué es un branch de hotfix?
Rama creada para corregir errores críticos en producción de forma rápida.

### ¿Qué es un release?
Versión específica y publicada de una aplicación o sistema.

### ¿Qué es versionado semántico?
Sistema de numeración de versiones basado en cambios mayores, menores y parches (ej: 1.2.3).

### ¿Qué es un breaking change?
Cambio en el software que rompe la compatibilidad con versiones anteriores.

### ¿Qué es una migración de base de datos?
Proceso de modificar la estructura o datos de una base de datos de forma controlada y reproducible.

### ¿Qué es un downtime?
Periodo en el que un sistema o servicio no está disponible para los usuarios.

### ¿Qué es blue/green deployment?
Estrategia de despliegue donde se mantienen dos entornos y el tráfico se cambia de uno a otro para minimizar downtime.

### ¿Qué es canary release?
Despliegue progresivo de una nueva versión a un subconjunto de usuarios antes de lanzarla a todos.

### ¿Qué es un health check?
Prueba automatizada para verificar que un sistema o servicio está funcionando correctamente.

### ¿Qué es un endpoint de status?
Ruta de una API que informa sobre el estado de salud del sistema.

### ¿Qué es un API Gateway?
Punto de entrada único para gestionar y enrutar peticiones a múltiples servicios backend.

### ¿Qué es un reverse proxy?
Servidor que recibe peticiones de clientes y las reenvía a uno o más servidores backend.

### ¿Qué es un edge server?
Servidor ubicado cerca del usuario final para reducir latencia y mejorar la entrega de contenido.

### ¿Qué es una política de backup?
Conjunto de reglas y procedimientos para realizar copias de seguridad y restaurarlas en caso de fallo.

### ¿Qué es disaster recovery?
Estrategia y procesos para restaurar sistemas y datos tras un desastre o fallo grave.

### ¿Qué es un incidente?
Evento que interrumpe o degrada el servicio de un sistema.

### ¿Qué es root cause analysis?
Proceso de investigación para identificar la causa raíz de un problema o incidente.

### ¿Qué es un postmortem?
Análisis retrospectivo de un incidente para aprender y evitar que se repita.

### ¿Qué es un playbook de incidentes?
Guía paso a paso para responder a incidentes específicos en un sistema.

### ¿Qué es un runbook?
Documento que describe procedimientos operativos rutinarios para sistemas y aplicaciones.

### ¿Qué es un dashboard de monitoreo?
Interfaz visual que muestra métricas y alertas clave sobre el estado de sistemas y aplicaciones.

### ¿Qué es Prometheus/Grafana/ELK Stack?
Herramientas populares para monitoreo, visualización y análisis de logs y métricas en sistemas modernos.

### ¿Qué es un alerting channel?
Canal de comunicación (Slack, email, SMS) donde se envían alertas automáticas de sistemas.

### ¿Qué es un secreto?
Información sensible como contraseñas, claves o tokens que debe protegerse y no exponerse en el código.

### ¿Qué es un vault?
Herramienta o servicio para almacenar y gestionar secretos de forma segura.

### ¿Qué es un token de acceso?
Credencial temporal que permite a un usuario o servicio acceder a recursos protegidos.

### ¿Qué es un refresh token?
Token que permite obtener un nuevo token de acceso sin volver a autenticarse.

### ¿Qué es un scope en OAuth2?
Permisos o alcances específicos que una aplicación puede solicitar al acceder a recursos de usuario.

### ¿Qué es un claim en JWT?
Atributo o dato incluido en el payload de un JWT, como el id de usuario o roles.

### ¿Qué es un rate limiter distribuido?
Sistema que aplica límites de peticiones a través de múltiples instancias o servidores.

### ¿Qué es un API key?
Clave única que identifica y autoriza a un cliente para consumir una API.

### ¿Qué es un CORS preflight?
Petición HTTP OPTIONS enviada por el navegador para verificar permisos antes de una petición real con CORS.

### ¿Qué es un CSP?
(Content Security Policy) Cabecera de seguridad que restringe los recursos que puede cargar una página web.

### ¿Qué es un SSRF?
(Server-Side Request Forgery) Vulnerabilidad donde un atacante puede hacer que el servidor realice peticiones a recursos internos.

### ¿Qué es un RCE?
(Remote Code Execution) Vulnerabilidad que permite a un atacante ejecutar código arbitrario en un servidor remoto.

### ¿Qué es un DoS/DDoS?
(Denial of Service/Distributed Denial of Service) Ataques que buscan saturar un sistema para dejarlo fuera de servicio.

### ¿Qué es un honeypot?
Sistema trampa diseñado para atraer y analizar ataques informáticos.

### ¿Qué es un pentest?
Prueba de penetración: simulación controlada de un ataque para identificar vulnerabilidades en un sistema.

### ¿Qué es un bug bounty?
Programa donde empresas recompensan a investigadores que reportan vulnerabilidades de seguridad.

### ¿Qué es un zero-day?
Vulnerabilidad desconocida para el fabricante y sin parche disponible.

### ¿Qué es un sandbox?
Entorno aislado donde se ejecuta código potencialmente peligroso sin afectar el sistema principal.

### ¿Qué es un entorno aislado?
Ambiente separado del entorno principal para pruebas, desarrollo o seguridad.

### ¿Qué es un contenedor efímero?
Contenedor que se crea y destruye rápidamente, usado para tareas temporales o pruebas.

### ¿Qué es un sidecar?
Patrón donde un contenedor adicional se ejecuta junto a la aplicación principal para añadir funcionalidades (ej: logging, proxy).

### ¿Qué es un operador de Kubernetes?
Extensión que automatiza la gestión de aplicaciones complejas en Kubernetes.

### ¿Qué es un CRD?
(Custom Resource Definition) Permite definir recursos personalizados en Kubernetes.

### ¿Qué es un pod?
Unidad mínima de despliegue en Kubernetes, que puede contener uno o más contenedores.

### ¿Qué es un deployment?
Objeto de Kubernetes que gestiona el despliegue y escalado de pods.

### ¿Qué es un service en Kubernetes?
Abstracción que expone un conjunto de pods como un servicio de red estable.

### ¿Qué es un ingress?
Objeto de Kubernetes que gestiona el acceso externo a servicios dentro del clúster.

### ¿Qué es un namespace?
Espacio lógico en Kubernetes para aislar recursos dentro de un clúster.

### ¿Qué es un configmap?
Objeto de Kubernetes para almacenar datos de configuración no sensibles.

### ¿Qué es un secret en Kubernetes?
Objeto de Kubernetes para almacenar información sensible como contraseñas o claves.

### ¿Qué es un volumen persistente?
Almacenamiento duradero en Kubernetes que sobrevive a la destrucción de pods.

### ¿Qué es un statefulset?
Objeto de Kubernetes para gestionar aplicaciones con estado y necesidades de identidad estable.

### ¿Qué es un daemonset?
Objeto de Kubernetes que asegura que una copia de un pod se ejecute en cada nodo del clúster.

### ¿Qué es un job/cronjob?
Job: ejecuta una tarea una vez. CronJob: ejecuta tareas programadas periódicamente.

### ¿Qué es un rolling update?
Despliegue progresivo de nuevas versiones de una aplicación sin downtime.

### ¿Qué es un rollback en Kubernetes?
Revertir un deployment a una versión anterior en caso de fallo.

### ¿Qué es un readiness/liveness probe?
Readiness: indica si un pod está listo para recibir tráfico. Liveness: indica si un pod está vivo o debe reiniciarse.

### ¿Qué es un autoscaler?
Componente que ajusta automáticamente el número de pods según la carga.

### ¿Qué es un pod disruption budget?
Restricción que limita cuántos pods pueden estar fuera de servicio durante tareas de mantenimiento.

### ¿Qué es un affinity/anti-affinity?
Políticas para controlar en qué nodos se programan los pods según afinidad o anti-afinidad.

### ¿Qué es un toleration?
Permite que un pod se ejecute en nodos con ciertas taints (marcas especiales).

### ¿Qué es un node pool?
Grupo de nodos con características similares dentro de un clúster de Kubernetes.

### ¿Qué es un cluster autoscaler?
Componente que ajusta automáticamente el tamaño del clúster según la demanda.

### ¿Qué es un cloud provider?
Proveedor de servicios en la nube como AWS, Azure o GCP.

### ¿Qué es IaaS/PaaS/SaaS?
Modelos de servicio en la nube: Infraestructura, Plataforma y Software como Servicio.

### ¿Qué es un VPC?
(Virtual Private Cloud) Red virtual privada dentro de un proveedor de nube.

### ¿Qué es un subnet?
Subred: segmento de una red más grande, usado para organizar y aislar recursos.

### ¿Qué es un firewall?
Sistema que controla el tráfico de red permitido o denegado según reglas definidas.

### ¿Qué es un NAT gateway?
Dispositivo que permite a recursos en una red privada acceder a internet usando una sola IP pública.

### ¿Qué es un bastion host?
Servidor seguro que actúa como punto de acceso a una red privada.

### ¿Qué es un load balancer layer 4/7?
Layer 4: balancea tráfico TCP/UDP. Layer 7: balancea tráfico HTTP/HTTPS y puede tomar decisiones basadas en contenido.

### ¿Qué es un sticky session?
Técnica para mantener a un usuario conectado siempre al mismo servidor durante una sesión.

### ¿Qué es un session affinity?
Sinónimo de sticky session: mantener la afinidad de sesión con un servidor específico.

### ¿Qué es un sticky cookie?
Cookie usada para implementar sticky sessions.

### ¿Qué es un balanceo round robin?
Estrategia de balanceo que distribuye peticiones de forma equitativa entre servidores.

### ¿Qué es un weighted load balancing?
Balanceo de carga que asigna más tráfico a servidores con mayor capacidad.

### ¿Qué es un health check endpoint?
Ruta especial para verificar la salud de un servicio.

### ¿Qué es un failover?
Cambio automático a un sistema de respaldo en caso de fallo del principal.

### ¿Qué es un cold/warm/hot standby?
Cold: respaldo apagado. Warm: respaldo encendido pero no activo. Hot: respaldo activo y sincronizado.

### ¿Qué es un RPO/RTO?
RPO: máximo de datos que se pueden perder. RTO: tiempo máximo para recuperar el servicio tras un fallo.

### ¿Qué es un DRP?
(Disaster Recovery Plan) Plan para restaurar sistemas tras un desastre.

### ¿Qué es un BCP?
(Business Continuity Plan) Plan para asegurar la continuidad del negocio ante incidentes graves.

### ¿Qué es un escalamiento automático?
Ajuste dinámico de recursos según la demanda del sistema.

### ¿Qué es un spot instance?
Instancia de computación en la nube ofrecida a bajo costo, pero que puede ser interrumpida por el proveedor.

### ¿Qué es un preemptible VM?
Máquina virtual de bajo costo que puede ser terminada por el proveedor en cualquier momento.

### ¿Qué es un rolling deployment?
Despliegue progresivo de una nueva versión reemplazando instancias antiguas gradualmente.

### ¿Qué es un canary deployment?
Despliegue de una nueva versión a un pequeño grupo de usuarios antes de lanzarla a todos.

### ¿Qué es un feature flag?
Técnica para activar o desactivar funcionalidades en producción sin desplegar nuevo código.

### ¿Qué es un dark launch?
Lanzamiento de una funcionalidad solo para un subconjunto de usuarios sin anunciarlo públicamente.

### ¿Qué es un A/B test?
Experimento donde se muestran dos versiones de una funcionalidad a diferentes usuarios para comparar resultados.

### ¿Qué es un shadow traffic?
Enviar tráfico real a una nueva versión del sistema en paralelo, sin afectar a los usuarios, para pruebas.

### ¿Qué es un chaos engineering?
Disciplina que busca mejorar la resiliencia de sistemas introduciendo fallos controlados y observando el comportamiento.

### ¿Qué es un fault injection?
Técnica para probar la resiliencia de un sistema introduciendo fallos artificiales.

### ¿Qué es un circuit breaker?
Patrón que previene llamadas repetidas a un sistema que está fallando, permitiendo su recuperación.

### ¿Qué es un bulkhead?
Patrón que aísla componentes para evitar que el fallo de uno afecte a los demás.

### ¿Qué es un retry policy?
Estrategia para reintentar operaciones fallidas según reglas definidas.

### ¿Qué es un backoff exponencial?
Técnica para aumentar el tiempo de espera entre reintentos de una operación fallida.

### ¿Qué es un dead letter queue?
Cola donde se envían mensajes que no pudieron ser procesados correctamente tras varios intentos.

### ¿Qué es un event sourcing?
Patrón donde el estado de una aplicación se construye a partir de una secuencia de eventos almacenados.

### ¿Qué es un CQRS?
(Command Query Responsibility Segregation) Patrón que separa las operaciones de lectura y escritura en sistemas complejos.

### ¿Qué es un message broker?
Sistema que gestiona el envío y recepción de mensajes entre aplicaciones o servicios.

### ¿Qué es un pub/sub?
(Publish/Subscribe) Patrón donde los emisores publican mensajes y los suscriptores los reciben sin conocer la fuente.

### ¿Qué es un event bus?
Canal centralizado para transmitir eventos entre diferentes partes de una aplicación.

### ¿Qué es un webhook?
Mecanismo para recibir notificaciones automáticas de eventos de otros sistemas mediante peticiones HTTP.

### ¿Qué es un polling?
Técnica donde un cliente consulta periódicamente a un servidor para saber si hay nuevos datos.

### ¿Qué es un long polling?
Variante de polling donde la petición se mantiene abierta hasta que hay nuevos datos o expira el tiempo de espera.

### ¿Qué es un SSE?
(Server-Sent Events) Protocolo para enviar actualizaciones en tiempo real del servidor al navegador mediante HTTP.

### ¿Qué es un websocket?
Protocolo que permite comunicación bidireccional y en tiempo real entre cliente y servidor.

### ¿Qué es un API rate limit?
Restricción sobre la cantidad de peticiones que un cliente puede hacer a una API en un periodo de tiempo.

### ¿Qué es un API quota?
Límite total de recursos o peticiones que un cliente puede consumir en una API.

### ¿Qué es un SLA breach?
Incumplimiento de un acuerdo de nivel de servicio (SLA).

### ¿Qué es un incident response?
Proceso y procedimientos para responder a incidentes de seguridad o fallos en sistemas.

### ¿Qué es un incident commander?
Persona responsable de coordinar la respuesta ante un incidente.

### ¿Qué es un post-incident review?
Análisis posterior a un incidente para identificar causas y mejoras.

### ¿Qué es un blameless postmortem?
Análisis de incidentes sin buscar culpables, enfocado en aprender y mejorar procesos.

### ¿Qué es un RCA?
(Root Cause Analysis) Proceso para identificar la causa raíz de un problema.

### ¿Qué es un SRE?
(Site Reliability Engineering) Disciplina que aplica principios de software a la gestión de sistemas y operaciones.

### ¿Qué es un error budget?
Cantidad máxima de errores permitidos en un periodo, usada para equilibrar innovación y estabilidad.

### ¿Qué es un toil?
Trabajo manual, repetitivo y sin valor añadido que debería ser automatizado.

### ¿Qué es un escalation policy?
Reglas que determinan cómo y a quién escalar alertas o incidentes según su gravedad.

### ¿Qué es un on-call?
Turno de guardia donde un ingeniero está disponible para responder a incidentes fuera del horario normal.

### ¿Qué es un paging?
Notificación urgente enviada a un ingeniero on-call para que atienda un incidente crítico.

### ¿Qué es un alert fatigue?
Cansancio causado por recibir demasiadas alertas, lo que puede llevar a ignorar notificaciones importantes.

### ¿Qué es un synthetic monitoring?
Monitoreo que simula el comportamiento de usuarios para detectar problemas antes de que afecten a usuarios reales.

### ¿Qué es un real user monitoring?
Monitoreo basado en la experiencia real de los usuarios finales.

### ¿Qué es un APM?
(Application Performance Monitoring) Herramienta para monitorear y analizar el rendimiento de aplicaciones.

### ¿Qué es un distributed tracing?
Técnica para rastrear una petición a través de múltiples servicios en una arquitectura distribuida.

### ¿Qué es un span?
Unidad básica de trabajo en un trace distribuido, representa una operación individual.

### ¿Qué es un trace?
Secuencia completa de spans que representan el recorrido de una petición a través de servicios.

### ¿Qué es un sampling?
Técnica para seleccionar una muestra representativa de datos para análisis, en vez de procesar todos los datos.

### ¿Qué es un log aggregation?
Proceso de recopilar y centralizar logs de múltiples fuentes para análisis y monitoreo.

### ¿Qué es un log shipping?
Envío de logs desde sistemas de origen a una plataforma centralizada para almacenamiento y análisis.

### ¿Qué es un log rotation?
Proceso de archivar y eliminar logs antiguos para gestionar el espacio de almacenamiento.

### ¿Qué es un log retention?
Política que define cuánto tiempo se conservan los logs antes de ser eliminados.

### ¿Qué es un SIEM?
(Security Information and Event Management) Plataforma para analizar y correlacionar eventos de seguridad en tiempo real.

### ¿Qué es un SOAR?
(Security Orchestration, Automation, and Response) Plataforma que automatiza la respuesta a incidentes de seguridad.

### ¿Qué es un SOC?
(Security Operations Center) Equipo responsable de monitorear y responder a incidentes de seguridad.

### ¿Qué es un incident response plan?
Plan formal que describe cómo responder a incidentes de seguridad.

### ¿Qué es un playbook de seguridad?
Guía detallada para responder a amenazas o incidentes de seguridad específicos.

### ¿Qué es un honeypot?
Sistema trampa diseñado para atraer y analizar ataques informáticos.

### ¿Qué es un deception technology?
Tecnología que engaña a los atacantes para detectar y analizar amenazas.

### ¿Qué es un threat intelligence?
Información sobre amenazas actuales y emergentes que ayuda a proteger sistemas y datos.

### ¿Qué es un IOC?
(Indicator of Compromise) Evidencia de que un sistema ha sido comprometido.

### ¿Qué es un TTP?
(Tactics, Techniques, and Procedures) Métodos usados por atacantes para comprometer sistemas.

### ¿Qué es un MITRE ATT&CK?
Base de datos de tácticas y técnicas usadas por atacantes, mantenida por MITRE.

### ¿Qué es un kill chain?
Modelo que describe las fases de un ataque informático desde el reconocimiento hasta la acción.

### ¿Qué es un red team/blue team/purple team?
Red team: simula ataques. Blue team: defiende. Purple team: colabora entre ambos para mejorar la seguridad.

### ¿Qué es un pentest?
Prueba de penetración: simulación controlada de un ataque para identificar vulnerabilidades en un sistema.

### ¿Qué es un bug bounty?
Programa donde empresas recompensan a investigadores que reportan vulnerabilidades de seguridad.

### ¿Qué es un zero trust?
Modelo de seguridad que asume que ninguna parte de la red es confiable por defecto.

### ¿Qué es un least privilege?
Principio de otorgar a los usuarios solo los permisos mínimos necesarios para realizar su trabajo.

### ¿Qué es un RBAC?
(Role-Based Access Control) Control de acceso basado en roles asignados a los usuarios.

### ¿Qué es un ABAC?
(Attribute-Based Access Control) Control de acceso basado en atributos de usuario, recurso y entorno.

### ¿Qué es un PBAC?
(Policy-Based Access Control) Control de acceso basado en políticas definidas por la organización.

### ¿Qué es un IAM?
(Identity and Access Management) Conjunto de procesos y tecnologías para gestionar identidades y accesos.

### ¿Qué es un MFA?
(Multi-Factor Authentication) Autenticación que requiere dos o más factores para verificar la identidad de un usuario.

### ¿Qué es un SSO?
(Single Sign-On) Sistema que permite a los usuarios acceder a múltiples aplicaciones con una sola autenticación.

### ¿Qué es un federated identity?
Modelo donde la identidad de un usuario es gestionada por un proveedor externo confiable.

### ¿Qué es un SCIM?
(System for Cross-domain Identity Management) Estándar para automatizar el intercambio de información de identidad entre dominios.

### ¿Qué es un provisioning?
Proceso de crear y configurar cuentas de usuario y recursos en un sistema.

### ¿Qué es un deprovisioning?
Proceso de eliminar cuentas de usuario y recursos cuando ya no son necesarios.

### ¿Qué es un JIT access?
(Just-In-Time Access) Acceso temporal concedido solo cuando es necesario y por tiempo limitado.

### ¿Qué es un break-glass account?
Cuenta con privilegios elevados usada solo en emergencias.

### ¿Qué es un privileged access management?
Gestión y control de cuentas con privilegios elevados para minimizar riesgos de seguridad.

### ¿Qué es un session hijacking?
Ataque donde un atacante toma control de una sesión activa de usuario.

### ¿Qué es un session fixation?
Ataque donde un atacante fija el identificador de sesión de un usuario antes de que inicie sesión.

### ¿Qué es un clickjacking?
Técnica que engaña a los usuarios para que hagan clic en elementos invisibles o disfrazados.

### ¿Qué es un open redirect?
Vulnerabilidad que permite redirigir a los usuarios a sitios maliciosos usando URLs manipuladas.

### ¿Qué es un SSRF?
(Server-Side Request Forgery) Vulnerabilidad donde un atacante puede hacer que el servidor realice peticiones a recursos internos.

### ¿Qué es un RCE?
(Remote Code Execution) Vulnerabilidad que permite a un atacante ejecutar código arbitrario en un servidor remoto.

### ¿Qué es un XXE?
(XML External Entity) Vulnerabilidad que permite a un atacante interferir con el procesamiento de XML.

### ¿Qué es un path traversal?
Vulnerabilidad que permite acceder a archivos fuera del directorio permitido usando rutas manipuladas.

### ¿Qué es un directory listing?
Configuración insegura que permite ver el listado de archivos de un directorio en un servidor web.

### ¿Qué es un insecure deserialization?
Vulnerabilidad donde datos deserializados pueden ejecutar código malicioso.

### ¿Qué es un insecure direct object reference?
Vulnerabilidad donde un usuario puede acceder directamente a recursos sin autorización adecuada.

### ¿Qué es un mass assignment?
Vulnerabilidad donde un atacante puede modificar campos no autorizados al enviar datos masivos a una API.

### ¿Qué es un insecure default configuration?
Configuración por defecto insegura que puede exponer el sistema a ataques.

### ¿Qué es un security misconfiguration?
Errores en la configuración de seguridad que pueden dejar el sistema vulnerable.

### ¿Qué es un hardcoded secret?
Clave o contraseña escrita directamente en el código fuente, lo que representa un riesgo de seguridad.

### ¿Qué es un sensitive data exposure?
Exposición accidental o intencionada de datos sensibles como contraseñas o información personal.

### ¿Qué es un insufficient logging & monitoring?
Falta de registros y monitoreo adecuados que dificulta la detección y respuesta a incidentes.

### ¿Qué es un broken access control?
Fallo en la implementación de controles de acceso que permite a usuarios realizar acciones no autorizadas.

### ¿Qué es un broken authentication?
Fallo en la autenticación que permite a atacantes suplantar identidades o acceder sin autorización.

### ¿Qué es un broken session management?
Fallo en la gestión de sesiones que puede permitir el secuestro o uso indebido de sesiones de usuario.

### ¿Qué es un insecure communication?
Transmisión de datos sin cifrado o protección adecuada, exponiendo la información a interceptación.

### ¿Qué es un insecure cryptographic storage?
Almacenamiento de datos sensibles sin cifrado o con algoritmos débiles.

### ¿Qué es un weak password policy?
Política de contraseñas que permite el uso de contraseñas fáciles de adivinar o poco seguras.

### ¿Qué es un brute force attack?
Ataque que intenta adivinar contraseñas probando muchas combinaciones posibles.

### ¿Qué es un credential stuffing?
Ataque que usa credenciales filtradas de otros servicios para intentar acceder a cuentas.

### ¿Qué es un password spraying?
Ataque que prueba una contraseña común en muchas cuentas diferentes.

### ¿Qué es un phishing?
Técnica de ingeniería social para engañar a usuarios y obtener información sensible.

### ¿Qué es un spear phishing?
Variante de phishing dirigida a una persona o grupo específico.

### ¿Qué es un whaling?
Phishing dirigido a altos ejecutivos o personas clave de una organización.

### ¿Qué es un vishing?
Phishing realizado a través de llamadas telefónicas.

### ¿Qué es un smishing?
Phishing realizado a través de mensajes SMS.

### ¿Qué es un social engineering?
Conjunto de técnicas para manipular a personas y obtener información confidencial.

### ¿Qué es un tailgating?
Acceso físico no autorizado a un área restringida siguiendo a una persona autorizada.

### ¿Qué es un shoulder surfing?
Observar a alguien mientras introduce información confidencial, como contraseñas.

### ¿Qué es un dumpster diving?
Buscar información confidencial en la basura o desechos de una organización.

### ¿Qué es un pretexting?
Crear una historia falsa para obtener información confidencial de una persona.

### ¿Qué es un baiting?
Ofrecer algo atractivo para engañar a una persona y obtener acceso o información.

### ¿Qué es un quid pro quo?
Ofrecer un beneficio a cambio de información confidencial o acceso.

### ¿Qué es un watering hole attack?
Comprometer un sitio web frecuentado por el objetivo para infectarlo con malware.

### ¿Qué es un drive-by download?
Descarga automática de malware al visitar un sitio web comprometido.

### ¿Qué es un malvertising?
Uso de anuncios maliciosos para distribuir malware o redirigir a sitios peligrosos.

### ¿Qué es un ransomware?
Malware que cifra los archivos de la víctima y exige un rescate para restaurarlos.

### ¿Qué es un spyware?
Software que recopila información sobre el usuario sin su consentimiento.

### ¿Qué es un adware?
Software que muestra anuncios no deseados al usuario.

### ¿Qué es un trojan?
Malware que se disfraza de software legítimo para engañar al usuario.

### ¿Qué es un worm?
Malware que se replica y propaga automáticamente a través de redes.

### ¿Qué es un virus?
Malware que se adjunta a archivos legítimos y se propaga al ejecutarlos.

### ¿Qué es un rootkit?
Herramienta que oculta la presencia de malware o actividades maliciosas en un sistema.

### ¿Qué es un botnet?
Red de dispositivos infectados controlados por un atacante para realizar ataques coordinados.

### ¿Qué es un C2 server?
(Comando y Control) Servidor usado por atacantes para controlar dispositivos comprometidos.

### ¿Qué es un DGA?
(Domain Generation Algorithm) Algoritmo usado por malware para generar nombres de dominio para comunicación con C2.

### ¿Qué es un fast flux?
Técnica para cambiar rápidamente las direcciones IP asociadas a un dominio malicioso y evadir detección.

### ¿Qué es un sinkhole?
Servidor que intercepta tráfico malicioso para analizarlo y evitar que llegue a su destino real.

### ¿Qué es un takedown?
Acción de desactivar o eliminar un recurso malicioso, como un dominio o servidor.

### ¿Qué es un takedown request?
Solicitud formal para eliminar un recurso malicioso.

### ¿Qué es un takedown notice?
Notificación legal para solicitar la eliminación de contenido o recursos.

### ¿Qué es un DMCA?
(Digital Millennium Copyright Act) Ley de derechos de autor en EE.UU. que regula la eliminación de contenido protegido.

### ¿Qué es un copyright strike?
Sanción aplicada por infringir derechos de autor en plataformas digitales.

### ¿Qué es un trademark infringement?
Uso no autorizado de una marca registrada.

### ¿Qué es un patent troll?
Persona o empresa que usa patentes de forma agresiva para obtener beneficios sin producir productos reales.

### ¿Qué es un cease and desist?
Carta legal que ordena a alguien detener una actividad ilegal o no autorizada.

### ¿Qué es un NDA?
(Non-Disclosure Agreement) Acuerdo de confidencialidad entre partes para proteger información sensible.

### ¿Qué es un MSA?
(Master Service Agreement) Contrato marco que regula la relación entre proveedor y cliente.

### ¿Qué es un SOW?
(Statement of Work) Documento que describe el trabajo a realizar en un proyecto.

### ¿Qué es un PO?

### ¿Qué es un invoice?

### ¿Qué es un purchase order?
Orden de compra emitida por un comprador a un proveedor.

### ¿Qué es un RFP?
(Request for Proposal) Solicitud formal de propuestas para un proyecto o servicio.

### ¿Qué es un RFQ?

### ¿Qué es un MOU?

### ¿Qué es un LOI?

### ¿Qué es un term sheet?
Documento que resume los términos clave de un acuerdo antes de firmar un contrato formal.

### ¿Qué es un cap table?
Tabla que muestra la distribución de propiedad y acciones en una empresa.

### ¿Qué es un vesting?
Proceso por el cual un empleado gana derechos sobre acciones o beneficios a lo largo del tiempo.

Periodo inicial en el que no se otorgan beneficios o acciones hasta que se cumple un tiempo mínimo.

### ¿Qué es un option pool?
